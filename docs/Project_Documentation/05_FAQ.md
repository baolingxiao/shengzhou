# 05_FAQ

> 基于当前仓库能力整理，含产品、技术、部署、安全、语音、记忆与运营问题。  
> 共 60 个常见问题。

## 通用问题

1. **这是什么产品？**  
   本项目是一个集聊天、记忆、语音、Agent 代办于一体的 AI 助手系统。

2. **它是网页版还是桌面版？**  
   核心是 PWA，可通过浏览器安装成“像桌面应用一样”的入口。

3. **支持本地部署吗？**  
   支持，默认脚本就是本地启动模式。

4. **支持服务器部署吗？**  
   支持，仓库提供 `deploy_server.sh`、systemd、Nginx 示例。

5. **需要联网吗？**  
   需要。模型调用、Realtime 语音、外部同步都依赖网络。

6. **可以离线使用吗？**  
   不是完整离线产品；部分本地能力可运行，但核心模型仍需 API。

7. **默认端口是多少？**  
   前端默认 `5190`，后端默认 `8766`。

8. **默认管理员账号是什么？**  
   默认读取 `.env` 的 `JARVIS_AUTH_USER/JARVIS_AUTH_PASSWORD`。

9. **可以注册普通用户吗？**  
   可以，前端启动页内置注册流程。

10. **多用户数据会混在一起吗？**  
    设计上按 `session_id=user-用户名` 隔离。

## 聊天与角色

11. **我能直接聊天吗？**  
    可以，登录后进入主界面即可发送消息。

12. **为什么登录后还要创建人物？**  
    普通用户首次登录必须完成 persona 配置后才能聊天。

13. **必须填 API Key 吗？**  
    普通用户 persona 至少需要填写一个 provider key。

14. **支持哪些模型提供方？**  
    当前主链路支持 Doubao 与 Claude；语音链路含 OpenAI Realtime。

15. **用户填的 API Key 会自动决定路由吗？**  
    当前代码保存了 key，但未完整接入多租户运行时路由。

16. **为什么回复风格会变化？**  
    受角色设定、信任度、工作模式、记忆上下文共同影响。

17. **能切换多个角色吗？**  
    角色存储支持多角色结构，但前端主要围绕默认“沈昼”设计。

18. **聊天历史会保存吗？**  
    会，短期记忆在会话过程中持续保存，并可归档。

19. **可以清空当前会话吗？**  
    后端支持 `/api/reset` 与管理端清空会话，前端普通界面无按钮。

20. **系统会显示“正在输入”吗？**  
    会，`thinking` 状态在 UI 中可见。

## 记忆系统

21. **记忆分几层？**  
    四层：规则层、长期记忆、短期工作记忆、瞬时记忆。

22. **长期记忆怎么存？**  
    同时写入 `knowledge_palace` 文件与 Chroma 向量库。

23. **为什么要双写？**  
    文件可读可审计，向量库用于语义检索。

24. **会自动做日结/周结吗？**  
    有维护服务与定时逻辑支持。

25. **我能手动触发维护吗？**  
    可以，开发者记忆面板支持 daily/weekly/monthly/catchup。

26. **能删除某几条记忆消息吗？**  
    可以，管理端支持按消息索引删除。

27. **普通用户能看到记忆后台吗？**  
    默认不能，入口是开发者专属按钮。

28. **有没有 Obsidian 同步能力？**  
    配置项存在，且记忆系统包含相关路径配置逻辑。

29. **记忆是不是一定每轮都写长期库？**  
    不是，有启发式和模型判定逻辑控制是否持久化。

30. **“记住这件事”会不会强制写入？**  
    有显式“强制写入长期记忆”的规则分支。

## 语音与实时

31. **支持语音输入吗？**  
    支持，点击麦克风可语音转文字。

32. **支持连续语音对话吗？**  
    支持，传统语音对话链路可持续监听与回复。

33. **支持实时语音吗？**  
    支持，`VITE_ENABLE_REALTIME_VOICE=true` 时走 WebRTC Realtime。

34. **为什么语音按钮是灰的？**  
    常见原因：后端离线、正在思考、麦克风权限未开、语音配置缺失。

35. **TTS 需要什么配置？**  
    至少需要 `ELEVENLABS_API_KEY` 与 `ELEVENLABS_VOICE_ID`。

36. **STT 默认用哪个？**  
    默认 `local`（faster-whisper）；也可切到 OpenAI 等 provider。

37. **唤醒词能改吗？**  
    能，通过 `NEURALPAL_VOICE_WAKE_PHRASES` 配置。

38. **Realtime 为什么连不上？**  
    先检查 `OPENAI_API_KEY`、网络连通性和浏览器麦克风权限。

39. **语音会不会自动播放很久？**  
    可随时关闭朗读或关闭语音对话。

40. **语音接口需要登录吗？**  
    当前 `/api/voice/*` 没有会话鉴权，生产环境需额外防护。

## Agent 与系统能力

41. **它会自己乱点电脑吗？**  
    设计上先 proposal，再用户确认，才会执行动作。

42. **可以取消待执行任务吗？**  
    可以，支持 pending cancel。

43. **执行中断后会丢任务吗？**  
    pending 任务会持久化到 `data/agent_sessions`。

44. **有风险控制吗？**  
    有 gate 风险分级与确认策略。

45. **工作模式是什么意思？**  
    角色按日程在“工作/陪伴/加班”间切换，影响可执行工具。

46. **加班会有什么影响？**  
    配置里定义了 overtime 成本（亲密度扣分）。

47. **macOS 权限为什么重要？**  
    Agent 代办依赖辅助功能、屏幕录制等系统权限。

48. **权限可以自动配置吗？**  
    支持“auto-setup”拉起授权流程，但仍需用户在系统界面确认。

49. **Windows/Linux 也有同等权限流程吗？**  
    当前权限引导重点实现在 Darwin（macOS）。

50. **更新是怎么做的？**  
    后端提供 Git 更新检测与应用更新接口，前端有更新提示模态。

## Shenzhou 集成与数据

51. **Shenzhou 是什么？**  
    外部世界引擎上下文系统，用于同步日数据与生活上下文。

52. **有哪些同步能力？**  
    用户日导出、life context 拉取、pipeline、主动触达、归档。

53. **前端能直接操作这些同步吗？**  
    当前无统一前端控制台，主要通过后端接口触发。

54. **开发模式能访问 `/world` 吗？**  
    Vite 仅代理 `/api`，`/world` 在开发态默认不可直接代理。

55. **有桥接鉴权吗？**  
    有，`/api/shenzhou/export-user-day` 与 `push-life-context` 使用内部 token。

## 安全、隐私、运维

56. **用户密码是加密存储吗？**  
    当前 `data/auth_users.json` 为明文，需上线前整改。

57. **会话令牌是 JWT 吗？**  
    是轻量 HMAC 签名 token，结构类似 JWT 但不是标准 JWT 库。

58. **聊天数据会上传到哪里？**  
    会发往你配置的模型服务商；本地也会存储记忆与 trace 文件。

59. **如何删除本地数据？**  
    可通过管理面板删除记忆，或手动清理 `data/` 下相关文件。

60. **如何反馈 Bug？**  
    建议提交可复现步骤 + 时间 + trace_id（若有）+ 后端日志。
